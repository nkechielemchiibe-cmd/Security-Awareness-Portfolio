 Phishing Incident Response Playbook

This playbook outlines the step‑by‑step process for responding to phishing incidents in a healthcare environment, ensuring rapid containment and protection of patient data.

 1. Detection
Phishing attempts may be reported by:
- Employees
- Automated email security tools
- IT helpdesk
- SOC monitoring alerts

 2. Initial Triage
- Verify if the email is malicious
- Identify affected users
- Determine whether any links were clicked or attachments opened
- Check if credentials were entered on a phishing site

 3. Containment
If a user clicked a link or opened an attachment:
- Disable the user’s account temporarily
- Block the malicious domain or sender
- Isolate affected devices from the network
- Reset passwords and enforce MFA re‑authentication

 4. Eradication
- Remove any malicious files
- Run endpoint scans
- Patch vulnerabilities if exploited
- Clear unauthorized sessions

 5. Recovery
- Restore normal account access
- Reconnect devices once clean
- Monitor for unusual activity for 24–48 hours

 6. User Education
- Notify the user of what happened
- Provide targeted phishing awareness training
- Reinforce reporting procedures

 7. Documentation & Reporting
- Record the incident details
- Update the incident log
- Report to compliance/privacy if PHI was at risk
- Review lessons learned

 8. Prevention Improvements
- Update email filters
- Add new phishing indicators to blocklists
- Improve staff training based on incident patterns
