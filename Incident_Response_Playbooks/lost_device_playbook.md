 Lost or Stolen Device Incident Response Playbook

This playbook outlines the steps for responding to lost or stolen devices (laptops, tablets, mobile phones) that may contain or access PHI or other sensitive healthcare data.

 1. Detection
A device may be reported lost or stolen by:
- Employees
- Clinical staff
- Security personnel
- IT asset management systems

 2. Initial Assessment
- Identify the type of device (laptop, phone, tablet)
- Confirm whether the device contains or can access PHI
- Determine if the device was encrypted
- Check whether MFA was enabled
- Identify the last known location or usage

 3. Containment
- Immediately disable the user’s account
- Revoke access tokens and remote sessions
- Trigger remote lock or remote wipe (if supported)
- Block the device from connecting to the network

 4. Eradication
- Remove the device from asset inventory as “lost/stolen”
- Reset passwords for all accounts associated with the device
- Review logs for suspicious access attempts

 5. Recovery
- Issue a replacement device
- Reconfigure access and MFA
- Restore necessary applications and permissions

 6. User Education
- Reinforce secure handling of devices
- Review physical security expectations
- Provide refresher training on PHI protection

 7. Documentation & Reporting
- Document the incident in the security log
- Notify privacy/compliance if PHI may be exposed
- File a report with physical security if theft is suspected
- Determine whether regulatory reporting is required

 8. Prevention Improvements
- Increase device encryption coverage
- Improve asset tracking
- Strengthen MFA enforcement
- Provide additional staff training on device security
